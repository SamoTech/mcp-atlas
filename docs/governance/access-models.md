# Governance: Access Models

## Read-only
Agents can search and retrieve information but cannot create, modify, delete, or trigger downstream operations.

## Write-confirm
Agents can prepare write actions, but execution requires explicit human confirmation.

## Admin-gated
High-impact actions such as deployment, bulk export, deletion, or permission changes require elevated approval and audit logging.

## Why this matters
Enterprises rarely fail on retrieval first. They fail when an agent can act too broadly with weak approval boundaries.
