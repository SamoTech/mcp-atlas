# Contributing to MCP Atlas

Thank you for helping make enterprise MCP adoption more transparent.

## What we need most

1. **Verified enterprise case studies** — the highest-value contribution
2. **Architecture pattern examples** backed by real deployments
3. **MCP server profiles** with enterprise-specific notes
4. **Corrections** to existing entries with evidence links

---

## Case Study Submission Standard

Every case study must include at minimum:

- **Company name** (no anonymization; unverifiable anonymous entries are rejected)
- **Industry**
- **Use case description** (what problem MCP solved)
- **Connected systems** (named tools, APIs, or data sources)
- **Architecture pattern** (link to one of our documented patterns)
- **Governance controls** (how access is managed)
- **Outcome evidence** (metrics, quotes, published references)
- **Source links** (public blog posts, talks, press releases)

Entries without source links receive a maximum score of ⭐⭐.

---

## Case Study Template

Copy [`data/schema/case-template.md`](data/schema/case-template.md) and place your file in `data/cases/company-name.md`.

---

## Scoring

The Enterprise Readiness Score is assigned by maintainers, not submitters. It is based solely on evidence quality. See the rubric in [README.md](README.md).

---

## Pull Request Process

1. Fork the repo
2. Create a branch: `case/company-name` or `pattern/pattern-name`
3. Fill in the template completely
4. Submit a PR with the source links in the description
5. A maintainer will review within 7 days

---

## Code of Conduct

This project follows the [Contributor Covenant](https://www.contributor-covenant.org/). Be direct, evidence-first, and respectful.
